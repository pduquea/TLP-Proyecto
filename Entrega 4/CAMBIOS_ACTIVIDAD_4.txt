1. CAMBIOS EN LA GRAMÁTICA BNF (gramaticas bnf/snake.bnf)
================================================================================

A. SOPORTE DE NIVELES DE DIFICULTAD
   - Añadido bloque: DEFINE LEVELS ... END
   - Nuevo token: LEVEL
   - Permite definir múltiples niveles (BABY, ENTHUSIAST, NYAN)
   - Cada nivel contiene propiedades: SPEED, POISON_FOOD, OBSTACLES, 
     INVULNERABILITY_DURATION, TAIL_COLORS

B. NUEVAS PROPIEDADES DE NIVEL
   - SPEED: Velocidad de tick (valor decimal, ej: 0.15, 0.10, 0.06)
   - POISON_FOOD: Cantidad de frutas venenosas habilitadas (0 o 1)
   - OBSTACLES: Cantidad de obstáculos fijos en el mapa
   - INVULNERABILITY_DURATION: Segundos de invulnerabilidad al comer powerup (7 segundos)
   - TAIL_COLORS: Array de colores hexadecimales para cola arcoíris en Nyan mode

C. SOPORTE DE NÚMEROS DECIMALES
   - Actualizada regla <numero>: ahora soporta formato "0.15" no solo enteros
   - Lexer usa patrón: \d+\.\d+|\d+ (decimales primero para precedencia)

D. NUEVOS EVENTOS
   - ON_EAT_POISON: Se dispara al comer fruta venenosa
   - ON_EAT_POWERUP: Se dispara al comer power-up de invulnerabilidad
   - ON_COLLISION_OBSTACLE: Se dispara al chocar con obstáculo

E. TIPOS DE FORMA (TYPE)
   - CIRCLE: Círculo (usado para cabeza Nyan, poison, powerup)
   - TRI / TRIANGLE: Triángulo (para formas alternativas)
   - BALL: Alias de CIRCLE
   - Por defecto: Rectángulo

3. CAMBIOS EN EL MOTOR DE JUEGO (runtime.py)
================================================================================

A. SISTEMA DE MENÚ DE NIVELES (GUI Tkinter)
   - Método: mostrar_menu_nivel()
   - Crea Frame overlay con Radiobuttons para seleccionar nivel
   - Niveles disponibles: BABY, ENTHUSIAST, NYAN
   - Se muestra antes de iniciar el juego
   - Selección es obligatoria para proceder

B. APLICACIÓN DE CONFIGURACIÓN DE NIVEL
   - Método: aplicar_config_nivel()
   - Lee propiedades del nivel seleccionado desde JSON
   - Aplica velocidad (velocidad_gravedad)
   - Habilita/deshabilita features (poison, powerup, obstacles)
   - Genera obstáculos aleatoriamente si es nivel NYAN
   - Carga colores de cola para modo Nyan

C. ATRIBUTOS NUEVOS DEL JUEGO SNAKE
   - self.levels: Diccionario de configuraciones de nivel
   - self.current_level: Nivel seleccionado (string)
   - self.level_config: Configuración del nivel actual (dict)
   - self.invulnerable: Boolean, indica si hay invulnerabilidad activa
   - self.invulnerable_tiempo: Timer de invulnerabilidad en segundos
   - self.invulnerability_duration: Duración configurada del nivel
   - self.poison_enabled: Si fruta venenosa está habilitada
   - self.obstacles_enabled: Si obstáculos están habilitados
   - self.powerup_enabled: Si powerup está habilitado
   - self.is_nyan_mode: Si nivel actual es NYAN
   - self.tail_colors: Array de colores para cola arcoíris
   - self.obstacle_positions: Lista de coordenadas de obstáculos
   - self.posicion_poison: Coordenada de fruta venenosa
   - self.posicion_powerup: Coordenada de powerup

D. LÓGICA DE INVULNERABILIDAD
   - Se activa al comer powerup (fruta amarilla)
   - Timer cuenta regresivamente: invulnerable_tiempo -= 0.05 por frame
   - Cuando timer llega a 0, se desactiva invulnerabilidad
   - Durante invulnerabilidad: colisiones con obstáculos restan 50 puntos pero no matan

E. FRUTAS VENENOSAS (Poison)
   - Color: #9933FF (morado)
   - Forma: CIRCLE (círculo)
   - Se spawnean aleatoriamente si poison_enabled=True
   - Al comerse: restan 10 puntos
   - GAME OVER: Solo si la puntuación BAJA A 0 desde un valor positivo
   - Si ya tienes 0 puntos y comes veneno: no ocurre nada especial, sigues con 0
   - Se respawnean automáticamente después de comerse
   - Nota: El evento ON_EAT_POISON se removió de snake.brick para usar handler por defecto

F. POWER-UP (Invulnerabilidad)
   - Color: #FFFF00 (amarillo)
   - Forma: CIRCLE (círculo)
   - Se spawnea siempre cuando está enabled (durante el juego)
   - Al comerse: activa invulnerabilidad por 7 segundos (configurable)
   - Se respawnea automáticamente después de comerse
   - Durante invulnerabilidad: colisiones con obstáculos restan puntos pero no matan
   - Nota: El evento ON_EAT_POWERUP se removió de snake.brick para usar handler por defecto

G. OBSTÁCULOS
   - Color: #555555 (gris oscuro)
   - Forma: RECTANGLE (por defecto)
   - Se generan aleatoriamente en nivel NYAN
   - Cantidad: configurable por nivel (ej: 5 obstáculos)
   - Al chocar: 
     * Si invulnerable: pierde 50 puntos
     * Si no invulnerable y puntos > 0: pierde 50 puntos
     * Si puntos llegan a 0 o menos: GAME OVER

H. MODO NYAN CAT
   - Se activa cuando TAIL_COLORS no está vacío
   - Cabeza: CIRCLE (círculo verde) con orejas de gato (triángulos grises)
   - Cola: Cada segmento usa color de array TAIL_COLORS (módulo índice)
   - Velocidad: 0.06 (más rápido)
   - Obstáculos: 5 generados aleatoriamente
   - Colisiones especiales: pierden puntos pero siguen vivos si puntos > 0
   - Colisiones con puntos = 0: GAME OVER

I. MÉTODO snake_spawn_poison()
   - Busca posición aleatoria sin colisiones
   - No colisiona con: cuerpo serpiente, comida normal, powerup, obstáculos
   - Asigna a self.posicion_poison

J. MÉTODO snake_spawn_powerup()
   - Busca posición aleatoria sin colisiones
   - No colisiona con: cuerpo serpiente, comida normal, poison, obstáculos
   - Asigna a self.posicion_powerup

K. MÉTODO snake_generar_obstaculos(cantidad)
   - Crea lista de obstáculos en posiciones aleatorias
   - Evita: cuerpo serpiente, comida, poison, powerup, otros obstáculos
   - Intenta 200 veces por obstáculo antes de desistir
   - Almacena en self.obstacle_positions

L. DIBUJO ACTUALIZADO (método dibujar)
   - Fruta venenosa: círculo morado (#9933FF)
   - Power-up: círculo amarillo (#FFFF00)
   - Obstáculos: rectángulos grises (#555555)
   - Modo Nyan: cabeza círculo verde + orejas triángulo + cola colores arcoíris
   - Segmentos body snake: uso de tail_colors[i % len(tail_colors)]

M. LOOP DE JUEGO (game_loop)
   - Timer de invulnerabilidad decrece cada frame (0.05 segundos)
   - Velocidad de tick controlada por velocidad_gravedad (del nivel)
   - Actualiza etiqueta de puntuación

N. MANEJO DE COLISIONES (snake_mover_jugador)
   - Pared (normal): GAME OVER
   - Pared (Nyan mode): Pierde todos los puntos, continúa si puntos > 0
   - Auto-colisión (normal): GAME OVER
   - Auto-colisión (Nyan mode): Pierde todos los puntos, continúa si puntos > 0
   - Comida: +10 puntos, crece, respawnea comida
   - Poison: -10 puntos, si puntos <= 0 GAME OVER, respawnea poison
   - Powerup: activa invulnerabilidad por duration segundos, respawnea
   - Obstáculo: -50 puntos (si invulnerable), si puntos <= 0 GAME OVER

O. EVENTOS POR DEFECTO (Handlers ejecutados si no hay evento personalizado en .brick)
   - ON_EAT_POISON: 
     * Guarda puntuación antes: puntuacion_antes = self.puntuacion
     * Resta 10 puntos: self.puntuacion -= 10
     * Si llega a negativo, setea a 0: if self.puntuacion < 0: self.puntuacion = 0
     * GAME OVER solo si bajó A cero: if puntuacion_antes > 0 and self.puntuacion == 0
     * Respawnea poison si está enabled
     * NOTA: Si ya tenías 0 puntos, comer veneno no causa game over
   
   - ON_EAT_POWERUP: 
     * Activa invulnerabilidad: self.invulnerable = True
     * Asigna timer: self.invulnerable_tiempo = 7 segundos
     * Respawnea powerup (siempre, no depende de flag)
   
   - ON_COLLISION_OBSTACLE: 
     * Si puntos <= 0: GAME OVER
     * Si puntos > 0: resta 50 puntos (a menos que esté invulnerable)
     * Si invulnerable: solo resta 50 puntos, no muere

9. FLUJO DE EJECUCIÓN COMPLETO

2. runtime.py carga JSON y crea instancia Juego con tipo='SNAKE'

8. game_loop() inicia:

   - Cada N ms (según velocidad_gravedad), llama snake_mover_jugador()
   - Detecta colisiones: comida, poison, powerup, obstáculos, paredes, self
   - Ejecuta eventos correspondientes
   - Dibuja canvas con todas las entidades
9. Si puntos <= 0 o colisión crítica: game_terminado = True
10. Fin del juego

PROBLEMA IDENTIFICADO:

- La fruta venenosa causaba game over incluso cuando ya la puntuación era 0

SOLUCIÓN IMPLEMENTADA:
1. Removido ON_EAT_POISON de snake.brick → Usa handler por defecto
2. Removido ON_EAT_POWERUP de snake.brick → Usa handler por defecto
3. Actualizado handler ON_EAT_POISON:
   - Verifica que puntuación BAJA A 0 (no si ya estaba en 0)
   - Respawnea poison automáticamente
4. Actualizado handler ON_EAT_POWERUP:
   - Activa invulnerabilidad (7 segundos)
   - Respawnea powerup automáticamente (siempre)

MEJORAS REALIZADAS:
✓ Power-up: Ahora se respawnea correctamente después de comerse
✓ Power-up: Invulnerabilidad activada correctamente por 7 segundos
